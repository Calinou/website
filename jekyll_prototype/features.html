Features

Intuitive and clean syntax.

Taking inspiration from multiple languages, the Nim syntax is easily understood and results in code that can be modified very efficiently.

  • Indentation significant syntax
  • Multiple constructs inspired by Python
  • Multi-line lambdas
  • Oberon-inspired visibility markers
  • Pascal-inspired type sections for leaner definitions
{% highlight nim %} import strutils type Person = object name*: string # Field is exported using `*`. age: Natural # Natural type ensures the age is positive. var people = [ Person(name: "John", age: 45), Person(name: "Kate", age: 30) ] for person in people: echo("$1 is $2 years old" % [person.name, $person.age]) {% endhighlight %}

Support for multiple operating systems

Nim supports every mainstream operating system and platform as well as multiple niche platforms. We take pride in supporting all three major operating systems fully.

Many garbage collector options.

Choose from a deferred RC'ing garbage collector that is fast, incremental and pauseless, a soft real-time garbage collector that is deterministic allowing you to specify its max pause time, and many others.

Source: https://forum.nim-lang.org/t/2646/3#18263

Native performance with state of the art optimisations

By compiling to C, Nim is able to take advantage of a wide range of mature C compilers. Because of this Nim is incredibly portable and is able to use all of the optimisations built into the toolchain.

The binaries produced by Nim have zero dependencies and are typically very small. This makes distribution of binaries easy and keeps your users happy.
